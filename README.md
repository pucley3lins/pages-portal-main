# pages-portal-main

## 项目简介

`pages-portal-main` 是一个用于归档与发布多个独立 HTML 页面的仓库。项目本身不绑定特定域名或网站，仅作为静态页面的集合与托管入口，方便统一管理与版本控制。

本仓库适合存放各类不依赖后端的轻量页面资源，如个人作品展示、文档页面、原型演示或临时页面归档等。

## 目录结构

```
pages-portal-main/
├── index.html          # 仓库入口页面（可自定义）
├── pages/              # 存放各独立 HTML 页面的文件夹
│   ├── page1/
│   ├── page2/
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
└── README.md           # 本文件
```

- `pages/`：每个子文件夹对应一个独立页面，内含该页面的 HTML 及相关资源。
- `assets/`：存放多个页面可共用的样式、脚本或图片，减少重复。

## 页面归档说明

- 每个页面应尽量保持独立，避免跨文件夹引用。
- 页面路径建议采用 `pages/页面名称/index.html` 格式，便于直接访问。
- 如需使用公共资源，请放置在 `assets/` 中，并在页面内通过相对路径引用。
- 页面内容不包含任何特定域名、营销信息或诱导点击元素。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在 `pages/` 下创建新文件夹，放入你的 HTML 页面及相关文件。
3. 如需修改入口，可编辑根目录下的 `index.html`。
4. 提交更改并推送至远程仓库，即可通过 GitHub Pages 或其他静态托管服务访问。

## 维护说明

- 本仓库由维护者定期整理，新增页面请遵循上述目录结构。
- 不建议在仓库内放置动态后端代码或大型文件（如超过 100MB）。
- 欢迎提交 Pull Request 添加新页面，但需确保内容符合通用规范（无违规、无广告）。
- 对于长期未更新或不再需要的页面，维护者可能会进行清理归档。
